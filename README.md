# SDKLiveness iOS SDK

## 🎯 ภาพรวมโปรเจค (Project Overview)

**SDKLiveness SDK** เป็น iOS Software Development Kit (SDK) ที่พัฒนาขึ้นมาเพื่อใช้สำหรับ **การยืนยันตัวตนด้วยใบหน้า (Liveness Detection)** โดยมีเป้าหมายหลักคือการตรวจสอบว่าผู้ใช้งานหน้ากล้องเป็น "คนที่มีชีวิตจริง" ไม่ใช่ภาพถ่าย วิดีโอ หรืองาน 3D (Spoofing)

SDK นี้ถูกออกแบบมาให้มีประสิทธิภาพสูง, ใช้งานง่าย, เป็นมิตรต่อความเป็นส่วนตัวของผู้ใช้งาน (รันโมเดลบนเครื่องทั้งหมด ไม่ส่งวิดีโอขึ้นเซิร์ฟเวอร์เพื่อประมวลผล) และปฏิบัติตามโครงสร้าง Clean Architecture เพื่อให้นำไปต่อยอดและดูแลรักษาได้ง่าย

## ✨ ฟีเจอร์หลัก (Key Features)

### 1. การตรวจสอบท่าทางแบบเรียลไทม์ (Active Liveness Detection)

ระบบจะสุ่มหรือสั่งให้ผู้ใช้งานทำท่าทางต่างๆ เพื่อตรวจสอบการตอบสนองแบบเรียลไทม์ รองรับทั้งหมด 4 ท่าทาง:

1. **หันซ้าย / ขวา (Turn Left/Right):** ตรวจจับการหมุนของศีรษะ (Yaw)
2. **กระพริบตา (Blink):** ตรวจจับการลืมตาและหลับตาผ่านอัตราส่วน Eye Aspect Ratio (EAR)
3. **ยิ้ม (Smile):** ตรวจจับความกว้างของรอยยิ้มเทียบกับใบหน้า (Lip Width Ratio)
4. **พยักหน้า (Nod):** ตรวจจับการก้มและเงยของศีรษะ (Pitch)

### 2. TrueDepth Anti-Spoofing (ป้องการการใช้รูปถ่าย)

- รองรับการใช้งาน **TrueDepth Camera (FaceID)** เพื่อดึงข้อมูลมิติความลึก (Depth Data) ของใบหน้า
- ใช้อัลกอริทึม **Double Curvature Check** ตรวจสอบความโค้งนูนของใบหน้าทั้งแนวตั้งและแนวนอน (จมูกต้องนูนกว่าแก้มและหน้าผาก)
- ป้องกันการนำรูปถ่ายแบนๆ (2D) หรือรูปถ่ายที่ถูกดัดโค้งมาสแกนได้อย่างเด็ดขาด โดยหากตรวจพบ ระบบจะบล็อคการสแกนและเนียนแจ้งเตือนผู้ใช้ว่า "ไม่พบใบหน้า กรุณาวางใบหน้าให้อยู่ในกรอบ" (Silent Rejection) เพื่อไม่ให้ผู้ไม่หวังดีรู้ตัวว่าระบบจับได้
- **Graceful Fallback:** หากใช้งานบนอุปกรณ์ที่ไม่มี TrueDepth ระบบจะทำงานต่อด้วยกล้องปกติอย่างราบรื่น

### 3. Apple Vision Framework (Zero 3rd-Party Dependencies)

- ระบบลบการพึ่งพาโมเดลภายนอกเช่น MediaPipe ออกทั้งหมด
- ใช้ **Apple Vision Framework** ที่มาพร้อมกับ iOS ซึ่งทำงานประสานกับ Neural Engine ของชิป Apple (A-Series) ได้อย่างสมบูรณ์แบบ
- ข้อดี: ประมวลผลเร็วมาก (30+ FPS), ไม่หนักเครื่อง, ประหยัดแบตเตอรี่, ไม่ต้องกังวลเรื่อง License ภายนอก

### 4. ระบบจัดการ Flow งานที่สมบูรณ์ (End-to-End UI Flow)

SDK มี UI ในตัวที่พร้อมใช้งานทันที ประกอบด้วย 10 หน้าจอหลัก:

- **Splash & License Check:** ตรวจสอบสิทธิ์การใช้งานของ SDK
- **Camera Permission:** ขอสิทธิ์การเข้าถึงกล้องอย่างถูกต้อง
- **Liveness Detection UI:** หน้าจอหลักที่มีกล้อง, กรอบวงรีแนะนำใบหน้า, แถบสถานะการทำท่าทาง (Progress), ข้อความแนะนำ, และแอนิเมชันลูกศรนำทาง
- **Result Screens:** หน้าต่างแสดงผลสำเร็จ, ล้มเหลว, หรือหมดเวลา (Timeout)

### 4. Clean Architecture & MVVM

- แบ่งแยกชั้นของโปรเจคชัดเจน (Core, Data, Domain, Presentation)
- มี Dependency Injection ที่ชัดเจน รองรับการเขียน Unit Test

## 🛠️ เทคโนโลยีที่ใช้ (Tech Stack)

- **Language:** Swift 5+
- **UI Framework:** SwiftUI (100%)
- **Camera:** AVFoundation (AVCaptureSession)
- **AI/ML:** Vision Framework (`VNDetectFaceLandmarksRequest`, `VNFaceObservation`)
- **Architecture:** Clean Architecture + MVVM + Coordinator Pattern

## 📝 Flow การทำงานของผู้ใช้งาน (User Journey)

1. แอปหลักที่เรียกใช้ SDK จะตั้งค่าจำนวนท่าทางที่ต้องการให้ผู้ใช้ทำ (เช่น 2 ท่าทางสุ่ม)
2. ผู้ใช้เห็นหน้า Splash Screen -> อนุญาตสิทธิ์กล้อง
3. ผู้ใช้เอาหน้าเข้ามาใน **"กรอบวงรี"** ระบบจะตรวจจับใบหน้า
4. ระบบสั่งให้ผู้ใช้ทำท่าทางที่ 1 (เช่น "กรุณายิ้ม") พร้อมแถบ Progress Bar
5. เมื่อผู้ใช้ทำถูกต้อง ระบบสั่งให้ทำท่าทางที่ 2
6. เมื่อทำครบทุกท่าทาง ระบบแสดงหน้าจอ "สำเร็จ" และคืนค่า `Session ID` หรือข้อมูลกลับไปยังแอปหลัก
7. หากทำไม่ทันเวลา หรือตรวจไม่พบใบหน้านานเกินไป ระบบจะแสดงหน้า "หมดเวลา" หรือ "ล้มเหลว"
